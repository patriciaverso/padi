***** FHS
Filesystem Hierarchy Standard. Gerenciador de serviços =systemd=.
****** =file-hierarchy(7)=
- =/=
  - file system root
- =/boot/=
  - EFI System Partition.
  - read-only.
- =/efi/=
  - ESP se não estiver no =/boot/=.
- =/etc/=
  - Configuração específica do sistema.
- =/home/=
  - home dos usuários normais.
  - não deve ser usado para usuários de sistema.
- =/root/=
  - home do root
- =/srv/=
  - "general server payload"
- =/tmp/=
  - small temporary files
  - "tmpfs"
- =/var/tmp/=
  - large temporary files
  - criar diretórios aqui só com =mkstemp(3)=, =mkdtemp(3)= e similares.
  - See =environ(7)= and =IEEE Std 1003.1[4]=
- =/run/=
  - "tmpfs"
  - runtime data
  - writable for privileged
  - =/run/log/=
    - runtime system logs
  - =/run/user/=
    - per-user runtime directories
    - "tmpfs"
    - See XDG Base Directory Specification[2]
- =/usr/=
  - vendor-supplied system resourcer
  - =/usr/bin/=
    - executables for user commands =$PATH=
    - invocation from a shell
  - =/usr/include/=
    - C and C++ API header files
  - =/usr/lib/=
    - static, private vendor data
    - public libraries on =$libdir=
  - =/usr/lib/arch-id/= (=$libdir=)
    - dynamic libraries.
    - not for package-specific data unless architecture-dependent
    - =# systemd-path system-library-arch= tells you the primary architecture
  - =/usr/share/=
    - resources shared between multiple packages
    - documentation, man pages, time zone information, fonts, others
    - =/usr/share/doc/=
      - operating system or system packages documentation
    - =/usr/share/factory/etc/=
      - vendor-supplied default configuration files
      - pristine versions of =/etc/=
    - =/usr/share/factory/var/=
      - vendor-versions of files in the variable, persistent data directory =/var/=
  - =/var/=
    - variable system data
    - =/var/cache/=
      - persistent system cache data.
      - rebuildable always
    - =/var/lib/=
      - persistent system data
    - =/var/log/=
      - persistent system logs
      - most logging should be done via =syslog(3)= and =sd_journal_print(3)=
    - =/var/spool/=
      - persistent system spool data, such as printer or mail queues.
    - =/var/tmp/=
      - larger and persistent temporary files.
      - usually mounted from a persistent physical file system
      - =mkstemp(3)= and =mkdstemp(3)=
  - =/dev/=
    - root directory for device nodes.
    - "devtmpfs"
    - managed by the kernel and =systemd-udevd(8)=
  - =/dev/shm/=
    - POSIX shared memory segments
    - =shm_open(3)=
  - =/proc/=
    - virtual kernel file system
    - process list
    - API to interface with the kernel
    - =/proc/sys/=
      - kernel tunables
      - settings in this API file tree should de configured via =sysctl.d(5)=
  - =/sys/=
    - =/sys/fs/cgroup/=
      - vkernel fs exposing process control groups.
  - =/bin/=
  - =/sbin/=
  - =/usr/sbin/=
    - symlinks pointing to =/usr/bin/= for compatibility
  - =/lib/=
    - points to =/usr/lib/=
  - =/lib64/=
    - points to =$libdir=
  - =/var/run/=
    - points to =/run/=
  - =~=
    - =~/.cache/=
      - user cache data
    - =~/.config/=
      - app configuration
    - =~/.local/bin/=
      - user-specific executables
      - invocation from console
    - =~/.local/lib/=
      - static, private vendor data compatible with all architectures
    - =~/.local/lib/arch-id/=
      - public dynamic libraries
    - =~/.local/share/=
      - resources shared between packages
    - =~/.local/state/=
      - application state

***** Boot the live environment
#+begin_src zsh  :results none
  # localectl list-keymaps
  # loadkeys colemak
  # setfont ter-132b
  # cat /sys/firmware/efi/fw_platform_size # tells you the boot mode (64 -> UEFI mode -> 64-bit x64 UEFI
  # ip link
  # iwctl station list
  # iwctl station wlan0 get-networks
  # iwctl --passphrase ... station wlan0 connect STARPLATINUMMESH
  # ping ping.archlinux.org
  # timedatectl set-ntp true
  # timedatectl # check if clock is synchronized
  # sgdisk --zap-all /dev/nvme0n1
  # sgdisk --new=1:0:+512M --typecode=1:EF00 --change-name=1:'EFI' /dev/nvme0n1
  # sgdisk --new=2:0:0 --typecode=2:8E00 --change-name=2:'LVM' /dev/nvme0n1
  # pvcreate /dev/nvme0n1p2
  # vgcreate archvg /dev/nvme0n1p2
  # lvcreate -L 4G archvg -n swap
  # lvcreate -L 30G archvg -n root
  # lvcreate -l 50%FREE archvg -n home
  # lvcreate -l 100%FREE archvg -n data
  # mkfs.fat -F32 /dev/nvme0n1p1
  # mkfs.ext4 /dev/archvg/root
  # mkfs.ext4 /dev/archvg/home
  # mkfs.nilfs2 /dev/archvg/data
  # mkswap /dev/archvg/swap
  # mount /dev/archvg/root /mnt
  # mkdir -p /mnt/boot/efi /mnt/home
  # mount /dev/nvme0n1p1 /mnt/boot/efi
  # mount /dev/archvg/home /mnt/home
  # mount /dev/archvg/data /mnt/data
  # swapon /dev/archvg/swap
#+end_src
Adicione =lvm2= na lista =HOOKS= em =/etc/mkinitcpio.conf=

Kernel 2.4.x+ contém um driver de console dividido em teclado e tela. O driver de teclado manda caracteres pra aplicação, que pede output pra tela pro driver de tela.
O pacote kbd complementa o driver de console.
